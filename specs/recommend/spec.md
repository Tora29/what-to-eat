# Feature: レコメンド (recommend)

## Overview

Cloudflare Workers AI（llama-3.1）を使って、カテゴリ別に料理をレコメンドする機能。
直近5件を除外し、手間（EASY / HARD）ごとに最適なメニューを最大3件ずつ選定して合計6件を返す。

API 詳細は [openapi.yaml](./openapi.yaml) を参照。

## User Stories

- ユーザーとして、「今日はメイン料理にしようか、おかずにしようか」を選んでレコメンドを受け取りたい。何を作るか迷ったときの助けにするため。
- ユーザーとして、最近作った料理がレコメンドから除外されてほしい。同じ料理が連続して提案されないようにするため。
- ユーザーとして、簡単な料理とめんどい料理をそれぞれ提案してほしい。その日の気分や時間に合わせて選べるようにするため。

## Schema

### レコメンドリクエスト（入力）

| フィールド | 型 | バリデーション |
|-----------|-----|---------------|
| category | `MAIN` \| `SIDE` | 必須 |

### レコメンドレスポンス（出力）

| フィールド | 型 | 説明 |
|-----------|-----|------|
| easy | `RecommendedDish[]` | EASY グループからの推薦（最大3件） |
| hard | `RecommendedDish[]` | HARD グループからの推薦（最大3件） |

### RecommendedDish

| フィールド | 型 | 説明 |
|-----------|-----|------|
| id | string | 料理 ID |
| name | string | 料理名 |
| recipeUrl | string \| null | レシピURL |
| tags | `{ id: string; name: string }[]` | タグ一覧 |

## API Endpoints

API 詳細は [openapi.yaml](./openapi.yaml) を参照。

| Method | Path | Description |
|--------|------|-------------|
| GET | /api/v1/recommend | カテゴリ別レコメンド取得 |

## 処理フロー

```
1. category（MAIN or SIDE）でレシピを全件取得
2. cookedAt 降順で並べ、直近5件を除外対象リストとする
3. 残りのレシピを effort（EASY / HARD）ごとに分類
4. 各グループを Workers AI（llama-3.1）に渡して最適な3件を選定
5. { easy: [...], hard: [...] } 形式で返す
```

## Acceptance Criteria

### 正常系

- AC-001: `category=MAIN` を指定してレコメンドを実行すると、MAIN カテゴリの料理から easy・hard 各最大3件（合計最大6件）が返される
- AC-002: `category=SIDE` を指定してレコメンドを実行すると、SIDE カテゴリの料理から easy・hard 各最大3件が返される
- AC-003: cookedAt 降順で直近5件に含まれる料理はレコメンド候補から除外される
- AC-004: レコメンド結果に料理名・レシピURL・タグが含まれる

### 異常系

- AC-101: 対象カテゴリの料理が0件の場合、`{ easy: [], hard: [] }` が返される（200 OK）
- AC-102: `category` パラメータが `MAIN` / `SIDE` 以外の場合、400 Bad Request が返される

### 境界値

- AC-201: 対象カテゴリの EASY 料理が直近除外後に2件しか残っていない場合、easy は2件返される
- AC-202: 対象カテゴリの HARD 料理が直近除外後に0件の場合、hard は空配列で返される
- AC-203: 登録レシピが5件以下の場合、直近5件の除外によって候補が0件になることがある（その場合は空配列を返す）

## UI Requirements

### 画面構成（/recommend）

- カテゴリ選択ボタン（メイン料理 / おかず）
- 「レコメンドする」ボタン
- レコメンド結果エリア
  - 「簡単」セクション（easy グループ、最大3件）
  - 「めんどい」セクション（hard グループ、最大3件）
  - 各料理カード: 料理名・タグ・レシピURLリンク

### インタラクション

- カテゴリ選択 → レコメンドボタンタップ → API 呼び出し → 結果表示
- ローディング中: スピナーを表示してボタンを無効化する
- レシピURLがある場合: 外部リンクで開く

### UX

- スマホ操作前提。カテゴリ選択は大きなトグルボタンで実装する
- レコメンド結果はスクロールせずに全件見えるようにカードをコンパクトにレイアウトする

## Non-Functional Requirements

### Performance

- レコメンド API のレスポンスは Workers AI の処理を含め 5000ms 以内を目標とする

### Security

- 認証済みユーザーのみアクセス可能（未認証は `/login` リダイレクト）

### Availability

- Workers AI が利用不可の場合でも 503 エラーを適切に返し、FE にエラーメッセージを表示する
