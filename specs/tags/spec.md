# Feature: タグ管理 (tags)

## Overview

料理に付与するタグのマスタ管理機能。専用の管理画面でタグの一覧表示・追加・削除を行う。
タグ名はシステム全体でユニーク。初期タグ10件はDBシードで登録済み。

API 詳細は [openapi.yaml](./openapi.yaml) を参照。

## User Stories

- ユーザーとして、タグ一覧を確認したい。登録されているタグを把握するため。
- ユーザーとして、新しいタグを追加したい。料理の分類を細かくするため。
- ユーザーとして、不要なタグを削除したい。使われなくなったタグを整理するため。

## Schema

### Tag エンティティ（出力）

> `packages/shared/src/schemas/tag.ts` の `tagSchema` を参照。

| フィールド | 型 | 説明 |
|-----------|-----|------|
| id | string (cuid) | 一意識別子 |
| name | string | タグ名（ユニーク） |
| createdAt | string (ISO 8601) | 作成日時 |

### タグ作成スキーマ（入力）

> `packages/shared/src/schemas/tag.ts` の `tagCreateSchema` を参照。

| フィールド | 型 | バリデーション |
|-----------|-----|---------------|
| name | string | 必須、1〜50文字 |

## API Endpoints

API 詳細は [openapi.yaml](./openapi.yaml) を参照。

| Method | Path | Description |
|--------|------|-------------|
| GET | /api/v1/tags | タグ一覧取得（name 昇順） |
| POST | /api/v1/tags | タグ新規作成 |
| DELETE | /api/v1/tags/:id | タグ削除 |

## Acceptance Criteria

### 正常系

- AC-001: タグ管理画面にアクセスすると、登録済みの全タグが name 昇順で一覧表示される
- AC-002: タグ名（1〜50文字）を入力して追加ボタンをタップすると、タグが登録されて一覧に即時追加される
- AC-003: タグの削除ボタンをタップすると、確認ダイアログなしにタグが削除されて一覧から即時消える

### 異常系

- AC-101: タグ名が空のまま追加ボタンをタップすると、「タグ名は必須です」エラーが表示されてタグは登録されない
- AC-102: 既に存在するタグ名を登録しようとすると、「同名のタグがすでに存在します」エラーが表示されてタグは登録されない（API: 409 Conflict）
- AC-103: 存在しない ID のタグを削除しようとすると、「タグが見つかりません」エラーが表示される（API: 404 Not Found）

### 境界値

- AC-201: タグ名がちょうど50文字の場合、正常に登録できる
- AC-202: タグ名が51文字以上の場合、「タグ名は50文字以内で入力してください」エラーが表示されてタグは登録されない

## UI Requirements

### 画面構成（/tags）

- ページタイトル「タグ管理」
- タグ追加フォーム（テキスト入力 + 追加ボタン）
- タグ一覧（各タグに削除ボタン付き）
- バリデーション / API エラーメッセージ表示エリア

### インタラクション

- 追加ボタンタップ: バリデーション実行 → API 呼び出し → 一覧更新またはエラー表示
- 削除ボタンタップ: 確認なしで即座に API 呼び出し → 一覧から削除
- 追加成功後: 入力フィールドをクリアする

### バリデーション表示

- name が空: 「タグ名は必須です」
- name が51文字以上: 「タグ名は50文字以内で入力してください」

### UX

- スマホ操作前提。削除ボタンは誤タップしにくい適切なサイズを確保する
- 追加フォームは画面上部に配置する

## Non-Functional Requirements

### Performance

- タグ一覧の初期ロードは 500ms 以内

### Security

- 認証済みユーザーのみアクセス可能（`/login` リダイレクト）
- タグ作成・削除は認証済みユーザーのみ実行可能（API: 401 Unauthorized）

### Accessibility

- ボタンに適切な `aria-label` を設定する
