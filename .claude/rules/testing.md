# Testing

テスト戦略と実装規約を定義するカテゴリ。

## このファイルに定義すべきこと

### テスト種別と境界

- Unit Test: テスト対象の範囲、モックの方針
- Integration / API Test: テスト対象の範囲、DB の扱い
- E2E Test: テスト対象の範囲、テスト環境の構成

### テスト-仕様連携

- AC（受入条件）とテストケースの紐付け形式（例: `[SPEC: AC-001]`）
- テストケース名の命名規約
- テストファイルの配置場所

### モック戦略

- 何をモックすべきか / すべきでないか
- モックライブラリの選定
- テストデータの管理方針

### カバレッジ方針

- カバレッジ目標（行 / ブランチ / 関数）
- カバレッジ計測ツール
- カバレッジレポートの活用方法

### テストコマンド

- 各テスト種別の実行コマンド
- Watch モード / 単発実行の使い分け
- CI での実行方法

## なぜ必要か

- scaffold-test-unit / scaffold-test-e2e スキルがテストコードを生成する際の規約
- test-and-fix スキルがテスト実行コマンドを判断する際の参照先
- spec-coverage スキルがカバレッジ分析する際の基準

## 参照するスキル

- scaffold-test-unit, scaffold-test-e2e, test-and-fix, spec-coverage
