# テスト実行と自動修正

テストを実行し、失敗した場合は原因を分析して修正する。

## 前提条件

- 実装コードが存在すること
- テストコードが存在すること

## ワークフロー

### 1. テストコマンドの判断

`specs/infra-spec.md` またはプロジェクトの設定ファイル（package.json, Makefile, pyproject.toml 等）からテスト実行コマンドを判断する。

### 2. テスト実行

判断したコマンドでテストを実行する。

### 3. 失敗時の分析

- エラーメッセージを確認
- スタックトレースから問題箇所を特定
- 期待値と実際の値の差分を分析

### 4. 修正の実装

失敗原因に応じて対処:

| 失敗原因             | 対処             |
| -------------------- | ---------------- |
| テストの期待値が古い | テストを更新     |
| 実装のバグ           | 実装を修正       |
| モックの設定ミス     | モック設定を修正 |

### 5. 再テスト

修正後、再度テスト実行。全て通るまで繰り返す。

### 6. E2E テスト（必要に応じて）

E2E テストコマンドが存在する場合は実行する。
