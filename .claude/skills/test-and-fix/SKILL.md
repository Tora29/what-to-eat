# テスト実行と自動修正

テストを実行し、失敗した場合は原因を分析して修正する。

## 前提条件

- 実装コードが存在すること
- テストコードが存在すること

## ワークフロー

### 1. テストコマンドの判断

`specs/infra-spec.md` またはプロジェクトの設定ファイル（package.json, Makefile, pyproject.toml 等）からテスト実行コマンドを判断する。

### 2. テスト実行

```bash
npm run test
```

### 3. 失敗時の分析

- エラーメッセージを確認
- スタックトレースから問題箇所を特定
- 期待値と実際の値の差分を分析

### 4. 修正の実装

**テストコードは仕様であるため、原則として編集しない。** 失敗原因に応じて対処:

| 失敗原因                       | 対処                                         |
| ------------------------------ | -------------------------------------------- |
| 実装のバグ                     | 実装を修正                                   |
| モックの設定ミス               | モック設定を修正                             |
| テストの期待値が仕様と食い違う | **実装を仕様に合わせる**（テストを変えない） |

テストの期待値を変更してよいのは、**ユーザーが明示的に仕様変更を指示した場合のみ**。

### 5. 再テスト

修正後、再度テスト実行。全て通るまで繰り返す。

### 6. E2E テスト（必要に応じて）

E2E テストコマンドが存在する場合は実行する。
