# アプリケーション検証

アプリケーションが正常に動作することを確認する。

## 前提条件（推奨）

- テストが通っていること（`/test-and-fix` 完了済み）

## ワークフロー

### 1. 検証コマンドの判断

`specs/infra-spec.md` またはプロジェクトの設定ファイル（package.json, Makefile, pyproject.toml 等）から以下の検証コマンドを判断する:

- 型チェック
- フォーマットチェック
- Lint チェック
- ユニットテスト / API テスト
- ビルド

### 2. 順次実行

判断したコマンドを以下の順序で実行:

1. **型チェック**: 型エラーがないことを確認
2. **フォーマットチェック**: コードフォーマットが適用されていることを確認
3. **Lint チェック**: コーディング規約違反がないことを確認
4. **テスト実行**: 全テストが通ることを確認
5. **ビルド**: 本番ビルドが成功することを確認

### 3. 開発サーバー起動（必要に応じて）

ブラウザでアクセス可能か確認する。

### 4. E2E テスト（必要に応じて）

E2E テストコマンドが存在する場合は実行する。

## エラー発生時

1. エラーメッセージを確認
2. 原因を特定
3. 修正を実施
4. 再度検証

全てのチェックが通るまで繰り返す。
